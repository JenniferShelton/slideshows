Well, now what?

Create your Software Carpentry workshop website

https://github.com/swcarpentry/workshop-template

We guide you through the process using git and GitHub.

Why do it this way?

  1. Allows instructors to customize material
  2. Ensures uniform appearance and metadata integrity
  3. Permits data to be accessible before, during, and after workshop
  4. Stresses git and GitHub experience is needed by at least one instructor

An overview

  1. Create and serve your workshop's website using GitHub Pages.
  2. Your repository: https://github.com/user/repo Your branch: gh-pages
  3. Your website is auto-magically served at http://user.github.io/repo/.
  4. Tech tips: Plain pages are published as-is. Pages with special headers are compiled.

Behind the Scenes

  • GitHub uses Jekyll to compile websites
  • Combines values from page's header...
  • ...and from a file called _config.yml...
  • ...with page templates stored in _layouts...
  • ...and snippets of HTML in _includes...
  • ...to create finished HTML pages

Our Conventions

  • Repositories are named YYYY-MM-DD-site
    • For example: 
      https://github.com/gvwilson/2015-07-01-mistaktonic
    • Website published at: 
      http://gvwilson.github.io/2015-07-01-miskatonic
  • We use data from the header of index.html to build the main website

There's a Catch

  • A user can only have one fork of a repo
  • But instructors teach many workshops
  • So we cannot simply fork workshop-template
  • Must import the template into a new repo

Steps

  1. Import
  2. Customize
  3. Check
  4. Publish

Step 1: Import

  1. Go to http://import.github.com
  2. Enter this URL: 
    https://github.com/swcarpentry/workshop-template
  3. Check it (GitHub won't import until you do)
  4. Select the owner and name
  5. Make sure it's public

Step 1: Import

Step 2: Customize

  • Go to your newly-created repository
  • Edit the header of index.html
  • Edit _config.yml to set lesson_repo and lesson_site
  • You can edit on GitHub or by cloning and editing on your desktop

Step 3: Check

  • Run tools/check.py to validate...
  • ...tools/preview to preview locally...
  • ...or tools/clean to clean up

4. Publish

  • Push to website
    • If you didn't simply edit there
  • Check your new website

Going Further

  • Add data files to your repository
  • Link to other sites
  • Create new pages (using the page layout)

Improving the Template

  • Fork workshop-template and send pull requests
  • Or file issues and we'll fix things
