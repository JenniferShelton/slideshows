January 2015

What

Create a website for a Software Carpentry workshop

See https://github.com/swcarpentry/workshop-template for full instructions

Why

  1. Let instructors make changes without editing the main website
  2. Uniform appearance and metadata
  3. Make sure at least one instructor knows how to use GitHub

How

  1. Create a brand new repository on GitHub
  2. Clone to desktop
  3. Pull content from workshop-template
  4. Edit and check
  5. Push to gh-pages branch

How

Why?

  • A user can only fork a repository once on GitHub
  • But instructors may teach many workshops

The Simple Way

  1. Download http://files.software-carpentry.org/swc-workshop-create
  2. swc-workshop-create YYYY-MM-DD-site "Workshop Title" your_github_id ...other instructors...
  3. Does steps 1-3 below

1. Create Repository

  • Not a fork of workshop-template
  • Name it YYYY-MM-DD-site
  • Can be owned by anyone

2. Clone to Desktop

git clone git@github.com:user/YYYY-MM-DD-site.git

3. Get Template Content

  1. git remote add template https://github.com/swcarpentry/workshop-template.git
  2. git checkout -b gh-pages
  3. git pull template gh-pages
  4. git remote rm template

Why gh-pages?

  • So GitHub will automatically create http://user.github.io/YYYY-MM-DD-site

4. Edit and Check

  1. Change the URLs in _config.yml
  2. Modify index.html (see below)
  3. Run tools/check.py to validate
  4. Can also run:
    • tools/preview to preview locally (requires Jekyll)
    • tools/clean to clean up

5. Push to Publish

  • git push origin gh-pages
  • Check http://user.github.io/YYYY-MM-DD-site

How the Magic Works

  • GitHub uses Jekyll to create websites
  • Combines key:value data from page header and _config.yml...
  • ...with page templates stored in _layouts...
  • ...and snippets for headers and footers stored in _includes...
  • ...to turn Markdown or HTML into finished pages

What to Edit

  • Change a few values in the header of index.html
  • Make sure the syllabus and schedule are right
  • Delete or uncomment setup instructions
  • See CUSTOMIZATION.md for full instructions

Going Further

  • Add data files to your repository
  • Link to other sites
  • Create new pages using the page layout

Improving the Template

  • Fork workshop-template and send pull requests as usual
    • Entries for FAQ are particularly welcome
  • File issues when anything goes wrong, gets in your way, seems confusing, ...
